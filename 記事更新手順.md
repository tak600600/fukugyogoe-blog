# 資産形成案内所|五右衛門 記事更新手順

## 📁 ファイル構成

```
fukugyogoe-blog/
├── posts/          ← ここに記事を追加する
│   ├── ai-gijiroku-45min.mdx
│   ├── shikaku-ai-study-plan.mdx
│   └── ikuji-15min-task.mdx
├── lib/
│   └── siteConfig.js  ← 数値（資産額・フォロワー）の更新はここ
└── public/            ← 画像ファイルはここ
```

---

## ✍️ 新しい記事を書く

### Step 1: ファイルを作成

`posts/` フォルダに `.mdx` ファイルを作成します。

ファイル名はURLになります（例: `ai-study-tip.mdx` → `/posts/ai-study-tip`）

### Step 2: 先頭にフロントマターを書く

```markdown
---
title: "記事タイトル（60文字以内推奨）"
date: "2026-04-15"
category: "ai"
excerpt: "記事の要約（TOPページのカードに表示される・100文字以内）"
author: "五右衛門"
---
```

**categoryの選択肢:**

| 値 | カテゴリ | 色 |
|---|---|---|
| `ai` | 🤖 AI活用 | 青 |
| `ikuji` | 👶 育児 | ピンク |
| `fukugyo` | 💼 副業 | 緑 |
| `money` | 📈 資産形成 | 黄 |
| `shikaku` | 📚 資格勉強 | 紫 |

### Step 3: 本文を書く

Markdownで書きます。

```markdown
## 見出し（h2）

本文テキスト。**太字**、*斜体*も使えます。

### 小見出し（h3）

- リスト項目1
- リスト項目2

> 引用文はこう書きます

コードブロック:
\```
コードをここに
\```
```

## 🔢 数値を更新する（資産額・フォロワー）

`lib/siteConfig.js` を開いて編集します：

```js
assetTracker: {
  current: 2000,   // ← 現在の資産額（万円）を変更
  target: 3000,
},
xFollowers: 182,   // ← Xのフォロワー数を変更
examDate: '2026-10-25',  // ← 試験日（変わったら変更）
```

## 🚀 GitHubにアップロード → 自動デプロイ

ファイルを編集・追加したら、**GitHub Desktopで変更をコミット・プッシュ**するだけ。  
Vercelが自動でビルドして、約2分でサイトに反映されます。

### GitHub Desktopの操作手順

1. GitHub Desktopを開く
2. 左側に「Changes」タブで変更ファイルが表示される
3. 下部の「Summary」に変更内容を一言入力（例: `記事追加: AI勉強法`）
4. 「Commit to main」ボタンをクリック
5. 右上の「Push origin」をクリック
6. 2〜3分後に `https://fukugyogoe.blog` を開いて確認

---

## 🖼️ 画像を使う場合

1. 画像を `public/images/` フォルダに入れる
2. 記事内で `![説明文](/images/ファイル名.jpg)` と書く

## ⚠️ よくあるミス

| ミス | 正しい書き方 |
|---|---|
| `date: 2026-04-15`（引用符なし） | `date: "2026-04-15"` |
| `category: AI活用` | `category: "ai"` |
| タイトルに`:`を使う | `title: "タイトル：サブタイトル"` のように引用符で囲む |
